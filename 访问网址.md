# 🌐 应用访问网址

## 本地访问

启动应用后，在浏览器中访问：

- **前端应用**: `http://localhost:8080`
- **后端 API**: `http://localhost:3001`
- **健康检查**: `http://localhost:3001/api/health`

## 局域网访问（同一 WiFi 下的设备）

如果你想在手机、平板或其他电脑上访问，可以使用你的 IP 地址：

### 查找你的 IP 地址

**macOS/Linux:**
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

**Windows:**
```bash
ipconfig
```

找到类似 `192.168.x.x` 或 `10.x.x.x` 的地址。

### 访问地址

- **前端应用**: `http://你的IP地址:8080`
  - 例如: `http://192.168.0.185:8080`
- **后端 API**: `http://你的IP地址:3001`
  - 例如: `http://192.168.0.185:3001`

## 快速获取访问地址

运行以下命令快速查看：

```bash
# 获取 IP 地址
IP=$(ifconfig | grep "inet " | grep -v 127.0.0.1 | head -1 | awk '{print $2}')

echo "本地访问:"
echo "  前端: http://localhost:8080"
echo "  后端: http://localhost:3001"
echo ""
echo "局域网访问:"
echo "  前端: http://$IP:8080"
echo "  后端: http://$IP:3001"
```

## 端口说明

- **8080**: 前端开发服务器（Vite）
- **3001**: 后端 API 服务器（Express）

## 注意事项

1. **防火墙**: 确保防火墙允许这些端口的访问
2. **同一网络**: 局域网访问需要设备连接到同一 WiFi 网络
3. **服务器运行**: 确保前端和后端服务器都已启动

## 测试连接

### 测试后端

```bash
curl http://localhost:3001/api/health
```

应该返回: `{"status":"ok"}`

### 测试前端

直接在浏览器打开 `http://localhost:8080`，应该看到应用首页。
